# 12 面试准备指南
> 把项目拆成“30 秒介绍、核心追问、关键数字和常见延伸”四个部分，方便对外表达。

## 本文目标

完成阅读后，你将理解：

1. 如何用 30 秒、2 分钟和 5 分钟介绍这个项目
2. AI 面试官和后端面试官最可能追问哪些点
3. 需要记住哪些关键数字

## 30 秒版本

“这是一个本地优先、可追溯、MCP 原生的 Agent 长期记忆引擎。Python 负责 SDK、提取和 MCP，Go 负责服务层、gRPC、认证和 SQLite 数据面。系统支持意图路由、多路检索、RRF 融合、冲突检测、遗忘策略和治理日志，同时提供测试、基准和部署文档。” 

## 2 分钟版本

可以从四个点展开：

1. **问题**：LLM Agent 跨会话会失忆，普通向量库又不足以支撑冲突、追踪和治理
2. **方案**：做一个 `SQLite + Go + Python + Protobuf` 的双语言记忆引擎
3. **能力**：支持嵌入模式和服务模式，提供 REST、gRPC、CLI、MCP 四类入口
4. **质量**：有 Python 测试、Go 测试、Go benchmark、Go/Python 对比脚本和 k6 压测脚本

## 5 分钟版本

可以按下面的顺序讲：

1. 为什么需要长期记忆
2. 为什么选择 SQLite
3. 为什么拆成 Go 数据面和 Python 智能面
4. 检索链路：意图路由 → 多路召回 → RRF → touch
5. 治理链路：冲突、遗忘、演化、审计、健康
6. 服务层：REST、gRPC、认证、中间件、可观测性
7. 质量保障：测试、基准、压测、CI

## AI 岗位高频问题

### “讲讲你的检索管线”

回答框架：

- 先做意图分类
- 根据意图选择 semantic / full-text / entity / causal trace
- 用 RRF 融合多路结果
- 最后刷新访问计数并返回 matched_by

### “为什么不用 LLM 做路由”

回答重点：

- 规则路由延迟更稳定
- 行为更可测
- 更适合写单元测试
- 当前问题空间仍可用关键词模式覆盖主要场景

### “遗忘算法怎么工作”

回答重点：

- 基于重要度、信任分、访问次数和时间衰减计算强度
- 长短期层使用不同的时间曲线参数
- 通过 `0.7 / 0.3` 双阈值控制升层和降层

### “如何处理冲突记忆”

回答重点：

- 先做相似度与词面重合筛选
- 再看正负极性和偏好类表达
- 命中后建立 `contradicts` 关系边
- 如有需要，可再接 LLM 复判

## 后端岗位高频问题

### “为什么用 Go”

- 服务层更适合静态二进制和并发处理
- gRPC、middleware、graceful shutdown 这些模式在 Go 里很成熟
- 可以把 Python 从服务端热路径里移开

### “gRPC 设计怎么做的”

- `StorageService` 定义 18 个 RPC
- `models.proto` 定义跨语言共享模型
- Python 端通过 `RemoteBackend` 调用 stub
- gRPC interceptor 统一处理认证

### “数据库怎么设计的”

- 核心有 6 张表
- Python 端额外有 `backend_meta`
- 15 个索引覆盖过滤、追踪和治理热点
- FTS5 负责全文检索，向量检索走 `sqlite-vec` 或余弦扫描

### “如何做优雅关停”

- 监听 `SIGINT` / `SIGTERM`
- 对 HTTP 使用 `Shutdown(ctx)`
- 对 gRPC 使用 `GracefulStop()`

## 常见追问

### “系统的扩展上限在哪里”

当前上限主要来自：

- 单节点 SQLite
- Go 侧向量检索仍是余弦扫描
- 多租户隔离还没落地

### “如果让你继续做两周，你会先做什么”

建议回答：

1. 优化 Go 向量检索路径
2. 完善多租户与认证授权
3. 把服务端治理任务做成可调度作业

## 必背数字

建议记住下面这些数字：

- 18 个 gRPC RPC
- 19 个 REST 操作
- 11 个 MCP 工具
- 6 张核心表 + Python 端 `backend_meta`
- 15 个常规索引
- 41 个既有测试基础上又补了一批 Go 测试
- Go benchmark 已覆盖存储、路由和编排
- LOCOMO-Lite 风格评估整体命中率 `50.0%`，baseline `23.3%`

## 面试表达建议

- 先讲问题，再讲方案
- 先讲边界，再讲细节
- 先讲取舍，再讲数字

这会让表达更有工程感，也更容易引出后续追问。

## 小结

- 这个项目既能从 AI 角度讲，也能从后端角度讲
- 检索、治理、协议和部署都是有效切入点
- 最好准备三种时长版本的介绍
- 关键数字一定要背熟
